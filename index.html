Search query expressions

In addition to searching beatmap artist / title / difficulty / mapper by text, you can use operators =, !=, < > <=, >= to compare value. For example, ar<=9 means AR less or equal to 9.

Values below are based on the current mod combination by default. You can specify another mod combination by writing mods_value, like hr_stars<5. The list of mod combinations are any, nm, ez, hr, dt, ezdt, hrdt, ezht, hrht.

  • stars: Star rating.
  • pp: PP for the perfect score.
  • length: Length in seconds.
  • bpm: BPM of the song.
  • ar: Approach rate.
  • cs: Circle size.
  • fc: A positive number represents the number of FCs with the current mods. A negative number represents the minimum number of misses with the current mods. Zero represents the beatmap had no scores with the current mods.
  • miss: Negation of fc.
  • fcmods: Bit flags representing whether there was an FC with the current mods plus additional mods (HD, FL). 2 for HD, 4 for FL, 8 for HDFL.
  • hdfc=0/1: Whether there was an FC with +HD or +HDFL.
  • flfc=0/1: Whether there was an FC with +FL or +HDFL.

Other values invariant to the mod combination:

  • status: 1 for Ranked, 2 for Approved, 4 for Loved.
  • mode: 0 for osu! (converted), 2 for osu!catch (specific).
  • date: Days since the beatmap is approved.
  • combo: Maximum combo number.
  • track: Track ID of a featured artist song. 0 for a non-featured song.
  • fa=0/1: Whether it is a featured artist song.

After a local data is loaded:

  • has=0/1: Whether the beatmap is present in the local data.
  • unplayed=0/1: Whether the beatmap is marked as unplayed in the local data.
  • played: Days since the last played date in the local data.
  • rank: 0 for SSH, 1 for SH, 2 for SS, 3 for S, 4 for A, ..., 9 for no data.
